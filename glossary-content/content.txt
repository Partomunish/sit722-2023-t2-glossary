
DevOps
An approach to software engineering that aims to align software developers and operations staff. In this way, there is nonconflicting tasking and a seamless transition of project artifacts via interoperable processes and tools.
Cois, C. A., Yankel, J., & Connell, A. (2014, October). Modern DevOps: Optimizing software development through effective system interactions. In 2014 IEEE international professional communication conference (IPCC) (pp. 1-7). IEEE.

Agile
An approach to software development that entails a set of values and principles that ensure the delivery of high-quality software products to customers through an adaptive, incremental, and iterative approach. It involves cross-functional and self-organized teams working collaboratively to achieve the desired outcomes.
Gheorghe, A. M., Gheorghe, I. D., & Iatan, I. L. (2020). Agile Software Development. Informatica Economica, 24(2).
Kuhrmann, M., Tell, P., Hebig, R., Klünder, J., Münch, J., Linssen, O., ... & Richardson, I. (2021). What makes agile software development agile?. IEEE transactions on software engineering, 48(9), 3523-3539.

Deployment
The process of releasing and installing software applications or updates onto target systems. Doing so makes the software operational and available for end-users.
Pressman, R. S. (2005). Software engineering: a practitioner's approach. Palgrave macmillan.

Release
The specific version or iteration of a software product that is made available to end-users or customers. It represents a stable and tested state of the software, ready for deployment in production or distribution to the target audience.
Pressman, R. S. (2005). Software engineering: a practitioner's approach. Palgrave macmillan.

Stakeholder
Any individual, group, or organization that has an interest or influence in the software project. Pressman, R. S. (2005). Software engineering: a practitioner's approach. Palgrave macmillan.

Cloud computing
A technology that allows users to access and use computing services, like storage and software, over the internet. Instead of relying on their own computers or servers, users can use remote resources hosted by providers.
Erl, T., Puttini, R., & Mahmood, Z. (2013). Cloud computing: concepts, technology & architecture. Pearson Education.
Alam, T. (2020). Cloud Computing and its role in the Information Technology. IAIC Transactions on Sustainable Digital Innovation (ITSDI), 1(2), 108-115.

Microservices Application
Software where a large application is broken down into smaller, independent services that can be developed, deployed, and managed separately. This approach aims to enhance flexibility, scalability, and maintainability of the overall system.
Newman, S. (2021). Building microservices. " O'Reilly Media, Inc.".
Davis, A. (2021). Bootstrapping Microservices with Docker, Kubernetes, and Terraform: A project-based guide. Simon and Schuster.

Microservice
A self-contained unit with its own specific functionality. It communicates with other microservices in the application through well-defined APIs.
Newman, S. (2021). Building microservices. " O'Reilly Media, Inc.".
Davis, A. (2021). Bootstrapping Microservices with Docker, Kubernetes, and Terraform: A project-based guide. Simon and Schuster.

Kubernetes
An open-source container orchestration platform used for automating the deployment, scaling, and management of containerized applications. It provides a robust system for coordinating and scheduling containers across a cluster of nodes. This allows applications to run consistently and reliably in various environments.
Hightower, K., Burns, B., & Beda, J. (2017). Kubernetes: Up and Running Dive into the Future of Infrastructure. OReilly Media. Inc., Sebastopol.
Davis, A. (2021). Bootstrapping Microservices with Docker, Kubernetes, and Terraform: A project-based guide. Simon and Schuster.

Docker
An open-source platform that enables developers to automate the deployment, scaling, and management of applications within containers. In this context, containers refer to lightweight, portable, and self-sufficient units that package the application's code and all its dependencies.
Poulton, N. (2023). Docker Deep Dive: Zero to Docker in a single book. NIGEL POULTON LTD.
Davis, A. (2021). Bootstrapping Microservices with Docker, Kubernetes, and Terraform: A project-based guide. Simon and Schuster.

Container
A lightweight, standalone, and executable software package that includes everything needed to run a piece of software. A container typically includes code, runtime, system tools, libraries, and settings.
Poulton, N. (2023). Docker Deep Dive: Zero to Docker in a single book. NIGEL POULTON LTD.
Zhang, Q., Liu, L., Pu, C., Dou, Q., Wu, L., & Zhou, W. (2018, July). A comparative study of containers and virtual machines in big data environment. In 2018 IEEE 11th International Conference on Cloud Computing (CLOUD) (pp. 178-185). IEEE.

Cluster
A group of interconnected computers or servers that work together to provide a higher level of performance, availability, and reliability. In DevOps, clusters are often used to manage and distribute containerized applications, as in the case of Kerbernetes clusters.
Hightower, K., Burns, B., & Beda, J. (2017). Kubernetes: Up and Running Dive into the Future of Infrastructure. OReilly Media. Inc., Sebastopol.

Virtual Machine
A software-based emulation of a physical computer. It allows multiple operating systems to run on a single physical machine, enabling greater resource utilization and isolation.
Smith, J. E., & Nair, R. (2005). The architecture of virtual machines. Computer, 38(5), 32-38.
Zhang, Q., Liu, L., Pu, C., Dou, Q., Wu, L., & Zhou, W. (2018, July). A comparative study of containers and virtual machines in big data environment. In 2018 IEEE 11th International Conference on Cloud Computing (CLOUD) (pp. 178-185). IEEE.

Image
A snapshot of a container or a virtual machine, capturing the application code, dependencies, and configurations. It serves as a template for creating consistent and reproducible environments.
Poulton, N. (2023). Docker Deep Dive: Zero to Docker in a single book. NIGEL POULTON LTD.

Continuous Integration
A development practice where code changes are frequently integrated into a shared repository. Each integration triggers automated testing and verification processes to ensure code quality and detect issues early.
Arachchi, S. A. I. B. S., & Perera, I. (2018, May). Continuous integration and continuous delivery pipeline automation for agile software project management. In 2018 Moratuwa Engineering Research Conference (MERCon) (pp. 156-161). IEEE.

Continuous Delivery
An extension of Continuous Integration, where code changes that pass automated testing are automatically deployed to production or staging environments. The aim here is to accelerate the delivery of software updates.
Arachchi, S. A. I. B. S., & Perera, I. (2018, May). Continuous integration and continuous delivery pipeline automation for agile software project management. In 2018 Moratuwa Engineering Research Conference (MERCon) (pp. 156-161). IEEE.

Behaviour-Driven Design
A software development approach that encourages collaboration between developers, quality assuarance, and non-technical stakeholders. The outcome of this collaboration is that the software is designed around the behaviour users are expected to demonstrate when using it.
Binamungu, L. P., Embury, S. M., & Konstantinou, N. (2018, March). Maintaining behaviour driven development specifications: Challenges and opportunities. In 2018 IEEE 25th International Conference on Software Analysis, Evolution and Reengineering (SANER) (pp. 175-184). IEEE.
Alhaj, M., Arbez, G., & Peyton, L. (2019). Approach of integrating behaviour-driven development with hardware/software codesign. Int. J. Innov. Comput. Inf. Control, 15(3), 1177-1191.

Test-Driven Design
A software development approach where developers write tests before writing the actual code. The aim here is to promote code quality, maintainability, and to ensure that the code meets its specified requirements.
Ekman, F., Johannesson, S., Peber, E., & Sandberg, C. Test-Driven Development: Drawbacks, Benefits, Industrial Usage and Complementary Methods.
Pombo, N., & Martins, C. (2021, July). Test Driven Development in Action: Case Study of a Cross-Platform Web Application. In IEEE EUROCON 2021-19th International Conference on Smart Technologies (pp. 352-356). IEEE.

Infrastructure as Code
A methodology that treats infrastructure provisioning, configuration, and management as software development tasks. It treats infrastructure components as code artifacts, allowing for consistency, version control, and reproducibility.
Sokolowski, D. (2022, November). Infrastructure as code for dynamic deployments. In Proceedings of the 30th ACM Joint European Software Engineering Conference and Symposium on the Foundations of Software Engineering (pp. 1775-1779).

Dependency
A software component or library that is required for a system to function correctly. Dependencies can be external libraries, modules, or services that are used by an application.
Pressman, R. S. (2005). Software engineering: a practitioner's approach. Palgrave macmillan.

Minimum Viable Product (MVP)
The first version of a software product that actually meets tyhe requirements of the customer. It has the minimum features required to satisfy early customers and gather feedback.
Mulder, J. (2021). Enterprise DevOps for Architects: Leverage AIOps and DevSecOps for secure digital transformation. Packt Publishing Ltd.

Definition of Done (DoD)
The first instance when a software product is really finished and meets the goals that have been set. There needs to be a clear and agreed-upon set of criteria that must be met for a product to be considered complete and ready for release.
Mulder, J. (2021). Enterprise DevOps for Architects: Leverage AIOps and DevSecOps for secure digital transformation. Packt Publishing Ltd.

Tier
A distinct level or layer within an application architecture or infrastructure. Tiers are often used to categorize and manage components with specific functionalities.
Tsui, F., Karam, O., & Bernal, B. (2022). Essentials of software engineering. Jones & Bartlett Learning.

Mean Time to Recovery (MTTR)
The average time required to restore a system, service, or application to full functionality after a failure or incident. In this way, it is a measure of the efficiency of incident response and recovery processes.
Mulder, J. (2021). Enterprise DevOps for Architects: Leverage AIOps and DevSecOps for secure digital transformation. Packt Publishing Ltd.
Tsui, F., Karam, O., & Bernal, B. (2022). Essentials of software engineering. Jones & Bartlett Learning.

Change Lead Time
The duration it takes for a code change or update following the latest release. It reflects the speed of the development and the overall efficiency of the team.
Mulder, J. (2021). Enterprise DevOps for Architects: Leverage AIOps and DevSecOps for secure digital transformation. Packt Publishing Ltd.
Tsui, F., Karam, O., & Bernal, B. (2022). Essentials of software engineering. Jones & Bartlett Learning.

Full Cycle Time
The duration of each iteration or each deployment. It relates to the time it takes for a user story or feature to move from the planning stage to being fully implemented, tested, and deployed into production.
Mulder, J. (2021). Enterprise DevOps for Architects: Leverage AIOps and DevSecOps for secure digital transformation. Packt Publishing Ltd.
Tsui, F., Karam, O., & Bernal, B. (2022). Essentials of software engineering. Jones & Bartlett Learning.

VOICE Model
An acronym for Value, Opportunity, Innovation, Customers, and Experience. It is a framework used to prioritize and evaluate the features of a software product by considering its potential impact and alignment with business goals.
Mulder, J. (2021). Enterprise DevOps for Architects: Leverage AIOps and DevSecOps for secure digital transformation. Packt Publishing Ltd.
Gheorghe, A. M., Gheorghe, I. D., & Iatan, I. L. (2020). Agile Software Development. Informatica Economica, 24(2).

NoSQL
A category of database systems designed for storing and retrieving large volumes of unstructured or semi-structured data. Unlike traditional relational databases, these databases offer flexible and scalable data models suitable for modern applications.
Connolly, T. M., & Begg, C. E. (2005). Database systems: a practical approach to design, implementation, and management. Pearson Education.

YAML
A recursive acronym for 'YAML Ain't Markup Language'. Overall, it is a human-readable data serialization format used for configuration files and data exchange between applications.
Eriksson, M., & Hallberg, V. (2011). Comparison between JSON and YAML for data serialization. The School of Computer Science and Engineering Royal Institute of Technology, 1-25.
Davis, A. (2021). Bootstrapping Microservices with Docker, Kubernetes, and Terraform: A project-based guide. Simon and Schuster.

RESTful Definition
Short for Representational State Transfer. It is a software architectural style for designing web applications. It emphasizes using standardized HTTP methods (GET, POST, PUT, DELETE) to interact with resources in a stateless and scalable manner.
Richardson, L., & Ruby, S. (2008). RESTful web services. " O'Reilly Media, Inc.".
Schreier, S. (2011, March). Modeling restful applications. In Proceedings of the second international workshop on restful design (pp. 15-21).

Terraform
An open-source Infrastructure as Code (IaC) tool that allows you to define and provision infrastructure resources like virtual machines, networks, and storage in a declarative configuration file. It then automates the provisioning and management of these resources across various cloud providers and platforms.
Brikman, Y. (2022). Terraform: Up and Running. " O'Reilly Media, Inc.".
Sindhu, G., & Pavithra, D. R. Deploying a Kubernetes Cluster with Kubernetes-Operation (kops) on AWS Cloud: Experiments and Lessons Learned.

Git
A distributed version control system (VCS) used for tracking changes in source code during software development. It enables multiple developers to collaborate on a project, tracks revisions, and facilitates the management of code branches and merges.
Brikman, Y. (2022). Terraform: Up and Running. " O'Reilly Media, Inc.".

Executable Documentation
Documentation that describes a system or process and contains code or scripts that can be executed to demonstrate or validate the described functionality. It is often used in DevOps to create living documentation that stays in sync with the actual system.
Brikman, Y. (2022). Terraform: Up and Running. " O'Reilly Media, Inc.".
Sindhu, G., & Pavithra, D. R. Deploying a Kubernetes Cluster with Kubernetes-Operation (kops) on AWS Cloud: Experiments and Lessons Learned.

Command-Line Interface (CLI)
A text-based interface used to interact with a computer program or operating system by entering commands and parameters via a command line. CLIs are often used in DevOps for tasks like configuring and managing software and infrastructure.
Brikman, Y. (2022). Terraform: Up and Running. " O'Reilly Media, Inc.".

Kubectl
A command-line utility used to interact with and manage Kubernetes clusters. It allows users to run commands to create, inspect, update, and delete Kubernetes resources.
Brikman, Y. (2022). Terraform: Up and Running. " O'Reilly Media, Inc.".

Kubernetes Node
A physical or virtual machine that serves as part of a cluster. Nodes run containerized applications managed by Kubernetes and provide the computing resources required to run those applications.
Salecha, R. (2022). Spring Boot App on AWS EKS. In Practical GitOps: Infrastructure Management Using Terraform, AWS, and GitHub Actions (pp. 263-321). Berkeley, CA: Apress.
Brikman, Y. (2022). Terraform: Up and Running. " O'Reilly Media, Inc.".

Kubernetes Pod
The smallest deployable unit in a Kubernetes cluster. It can contain one or more containers that share the same network and storage resources. In this way, pods are used to deploy and scale containerized applications.
Salecha, R. (2022). Spring Boot App on AWS EKS. In Practical GitOps: Infrastructure Management Using Terraform, AWS, and GitHub Actions (pp. 263-321). Berkeley, CA: Apress.
Brikman, Y. (2022). Terraform: Up and Running. " O'Reilly Media, Inc.".

HashiCorp Configuration Language (HCL)
A domain-specific language used for writing configuration files that define Infrastructure as Code. It is commonly used with HashiCorp tools such as Terraform and Consul.
Sabharwal, N., Pandey, S., & Pandey, P. Infrastructure-as-Code Automation Using Terraform, Packer, Vault, Nomad and Consul.

Man-in-the-Middle
A cyberattack where an attacker intercepts and potentially modifies communications between two parties without their knowledge. These attacks can be used to eavesdrop on or manipulate data transmitted between systems.
Chuka-Maduji, N., & Anu, V. (2021). Cloud computing security challenges and related defensive measures: A survey and taxonomy. SN Computer Science, 2(4), 331.

Live Reload
A development feature that automatically refreshes or reloads a web application's user interface in real-time as code changes are made. It is commonly used to accelerate the development and testing of web applications.
Williams, L., Beck, K., Creasey, J., Glover, A., Holman, J., Humble, J., ... & Parnin, C. (2020). Continuous Deployment Transitions at Scale. In Tools and Techniques for Software Development in Large Organizations: Emerging Research and Opportunities (pp. 168-181). IGI Global.
